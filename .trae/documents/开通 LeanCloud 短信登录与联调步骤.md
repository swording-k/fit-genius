## 你需要完成的操作
- 注册 LeanCloud 并创建应用（国内版）：完成企业/个人实名认证。
- 开通短信服务：配置短信签名与模板，提交审核。
- 获取并记录三项配置：`AppID`、`AppKey`、`REST API 域名（baseURL）`。
- 在 Xcode 的 `Info.plist` 添加键值：
  - `LEAN_APP_ID` = 你的 AppID
  - `LEAN_APP_KEY` = 你的 AppKey
  - `LEAN_BASE_URL` = 例如 `https://api.leancloud.cn`
- 准备短信测试手机号（你自己的手机），确保可接收短信。

## 详细步骤
### 1. 注册与创建应用
- 访问 LeanCloud 官网，注册并实名认证（短信与签名需要实名认证后可用）。
- 控制台 → 创建应用（选择国内节点），记住应用的 `AppID` 与 `AppKey`。
- 控制台 → 设置 → 应用 Key，确认有 REST API 访问域名（通常国内是 `https://api.leancloud.cn`）。

### 2. 申请短信签名与模板
- 控制台 → 短信服务 → 短信签名：新增签名（例如“FitGenius”），提交审核材料。
- 控制台 → 短信服务 → 模板：新增“登录验证码”模板，示例内容：
  - 模板示例：`验证码：{code}，用于登录，5分钟内有效。`（按平台规范填写变量名）
- 待签名与模板审核通过（通常几十分钟到1个工作日）。

### 3. 配置白名单与限速（可选但推荐）
- 控制台 → 短信服务 → 安全设置：启用频控（如同一手机号 60 秒一次、每日上限）。
- 添加你的测试手机号到白名单，便于联调期间稳定收信。

### 4. 将配置填入工程
- 打开 Xcode，选中 `Info.plist`：添加 3 条字符串项：
  - `LEAN_APP_ID`: 你的 LeanCloud AppID
  - `LEAN_APP_KEY`: 你的 LeanCloud AppKey
  - `LEAN_BASE_URL`: 你的 REST 域名（国内一般 `https://api.leancloud.cn`）
- 无需修改 ATS（LeanCloud 为 HTTPS）。若使用自定义域名，需确保启用 TLS。

### 5. 联调与验证
- 在 App 的“设置”页（已内置）：
  - 输入手机号 → 点击“获取验证码”（按钮会开始 60s 倒计时）
  - 收到短信后输入验证码 → 点击“手机号登录”
  - 登录成功后顶部显示“已登录”，并写入 `UserProfile.userId` 与会话（Keychain）
- 若报错：设置页会显示错误描述；我可据错误码指导进一步在控制台或工程调整。

### 6. 云同步（可选但推荐）
- Xcode → Targets → Signing & Capabilities：添加 `iCloud` 并启用 `CloudKit`（选择默认容器）。
- 设置页执行“手动上传到云端”与“从云端拉取最新”，确认能在多设备间同步训练计划。

### 7. 安全与隐私
- AppKey 不要提交到公共仓库；当前工程从 `Info.plist` 读取，保持安全。
- 会话令牌存储在 Keychain；退出登录会清除。
- 后续可添加“注销账号与数据删除”入口以满足合规。

### 8. 我需要你提供/确认的内容
- LeanCloud：`AppID`、`AppKey`、`REST 域名`（发给我即可，我会配置到本地并联调）。
- 短信签名与模板审核通过的状态（即可开始真实短信联调）。
- 是否启用 CloudKit 容器（如果需要多设备同步）。

- 登录后在“云同步”分区可以上传/拉取；拉取成功后计划在另一设备可见。
